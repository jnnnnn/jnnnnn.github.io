GitGuardian


This tool applies if:

  • You write code, and push that code to GitHub or GitLab

  • You create Docker images, and push those images to Artifactory


GitGuardian is installed by default on all of our repositories. An engineer should use org-repo-settings to configure this as a required status check on all pull requests.


  • It’s a tool that runs on source code repositories

  • It aims to detect secrets in our code base and docker images and improve our security posture.



  • Credentials that could be used to connect to various internal or external systems might be exposed, leading towards hacking of systems, outages, loss of productivity, loss of trust, and significant financial penalties for Iress

  • Engineers wasting time rotating, revoking and remediating leaked secrets



Developer experience - tooling


  • It’s built in to docker-image-builder

  • Less than 5 minutes to install the tool as a pre-push hook on your development machine

  • Less than a few seconds each time you run "git push"

  • Less than 5 minutes to configure GitGuardian as a Required Status check on all of your oneiress and oneiresslab repository if you've set up org-repo-settings (iress-io repositories have this done automatically)
