SonarQube


This tool applies if you write application code.


It should be running and reporting back on all Pull Requests as a required status check, and scanning your main branch when code is merged.


Results should be monitored to ensure detected security hotspots, as well as “blocker”, “critical”, and “high” bugs and vulnerabilities are addressed within a timely manner.


SonarQube is a Static Analysis tool. It runs against application codebases to detect potential bugs, security hotspots, and application security issues. 

It helps prevent poor quality or vulnerable code from being accepted into the codebase. 



Poor quality code, bugs, and vulnerabilities could go undetected, resulting in:

  • Increased code maintenance costs for the product engineering team

  • An inability for teams to assess risks and prioritise remediation

  • Failing compliance audits, or exposing Iress to various cyber attacks leading to reputational damage for Iress



Developer experience - tooling


  • Less than 1 hour to configure as per the user guide

  • A variable amount of time to run as part of your build pipeline, depending on the number of lines of code and programming language in use
