org-repo-settings


This tool applies if you own or maintain repositories on the oneiress GitHub organisation. 


It should be configured by engineering teams for each repository that contains application or infrastructure code, in order to ensure our repositories comply with GitHub Repository / Branch guardrails and compliance audits.



org-repo-settings is an internally developed tool that: 

  • Configures repository settings via config-as-code

  • Helps teams meet the GitHub Repository / Branch Guardrails




  • Code may be able to be merged to main without peer review, leading to compliance audit failures resulting in reputational damage for Iress

  • Bad actors who have compromised our GitHub accounts may be able to make code changes unnoticed, exposing Iress to various cyber attacks





Developer experience - tooling


  • Less than 1 hour to add a configuration file to org-repo-settings

  • Less than 10 minutes to make minor adjustments to existing configuration files
