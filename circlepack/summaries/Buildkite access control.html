buildkite-access-control


This tool applies if you use Buildkite deploy agents to deploy your applications or infrastructure.


It needs to be put in place by engineering teams for each repository that code is deployed from.





Buildkite access control is an internally developed tool that restricts which GitHub repositories and branches are allowed to deploy from Buildkite agents that service line-of-business AWS accounts.



  • Accidental production environment destruction could occur, causing pain for engineering teams

  • Compromised employee accounts may exploit or destroy production environments leading to reputational damage or fines for Iress

  • Failing compliance audits leading to reputational damage for Iress


Developer experience - tooling


  • Less than 4 hours to add a new line of business configuration file to buildkite-access-control

  • Less than 10 minutes to make minor adjustments to existing configuration files
