Dependabot


This tool applies if you maintain GitHub repositories that contain third party dependencies.


Dependabot is enabled automatically on all repositories in all of our GitHub organisations. 


Engineering teams should be monitoring its output and responding to security alerts and dependency update recommendations. 


It’s a Dependency maintenance tool that: 

  • Alerts if insecure dependencies are in use

  • Performs dependency maintenance




  • Vulnerabilities go unnoticed and are introduced into our applications exposing Iress to various cyber attacks

  • Dependencies get outdated resulting in code health deterioration and increased uplift effort for product engineering teams

  • Failing compliance audits leading to reputational damage for Iress




Developer experience - tooling


  • Less than 1 hour to configure team based security alerts via dependabot-alerts-collector

  • Less than 1 hour to configure dependency update rules, if desired
