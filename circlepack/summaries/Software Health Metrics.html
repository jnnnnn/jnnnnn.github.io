Software Health Metrics


The tool applies to:

  • Engineers creating “Application” Software in the oneiress GitHub organisation.

  • Engineers maintaining software developed by Iress, whose code exists in another version control system (e.g TFS).


An assessment of metrics measuring software health, giving us an Iress-wide baseline of software health against the key capabilities known to result in improved speed, scale and quality.

Supports software owners in highlighting opportunities for improving their software health.


Delivery Coach


  • Most squads organise a half-day workshop to explore the Software Health Metric questions, debate the assessment for each metric and create an initial assessment file.

  • Most squads organise a two-hour workshop every three months to review the Squad Health Metric questions and reevaluate the assessments for the squad.

  • Most squads use their Retrospective meetings to brainstorm and agree experiments to improve their health metrics. These sessions should take no more than three hours every month.
