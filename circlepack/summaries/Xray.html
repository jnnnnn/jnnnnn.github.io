JFrog Xray


This tool applies if:

  • You create docker containers for build or deployment purposes

  • You create applications that contain third party dependencies


Xray should run as part of pull request workflows, and the Product Engineering teams should address any detected vulnerabilities or non-compliance issues as part of the pull request workflow. 


It’s a Software Composition Analysis (SCA) tool that: 

  • Scans docker images and application packages for vulnerable components

  • Scans application packages to ensure third party license compliance




  • Vulnerabilities in our docker containers or applications exposing Iress to various cyber attacks

  • Legal action against Iress due to non-compliance with 3rd party license conditions

  • Failing compliance audits leading to reputational damage for Iress




Developer experience - tooling


  • It’s built in to docker-image-builder

  • Less than 10 minutes to add an iress-xray-scanner build step to your build pipeline

  • A variable ongoing amount of time remediating existing or newly detected vulnerabilities
