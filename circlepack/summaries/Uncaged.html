Iress Uncaged 


The tool applies to:

  • Engineers creating “Application” Software in the oneiress GitHub organisation.

  • Engineers maintaining software developed by Iress, whose code exists in another version control system (e.g TFS).



Uncaged uses the data provided in the Atlas registry to gather metrics and present metrics.
For software owners this tool provides a central location to review key aspects of their software (security, cost, quality, health). Uncaged will provide a set of prioritised recommendations on how to improve the health of the software.


Uncaged provides a convenient means to review governance & quality expectations for a team operating software at Iress. Those who do not provide the necessary information for Uncaged to gather this data, will need to perform these governance & quality checks in the eight different tools this data is aggregated from.


Engineering Enablement


Uncaged requires you to register on Atlas, by committing a yaml file into a GitHub repository. The Atlas yaml file has a defined specification that will be validated in your PR.


Gathering the information to populate the Atlas yaml file should take less than four hours. Making changes to the yaml should take under 10 minutes. 


The data in the Uncaged dashboard is refreshed every two hours.
