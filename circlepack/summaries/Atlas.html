Iress Atlas 


The tool applies to:

  • Engineers creating code repositories in the oneiress GitHub organisation.

  • Engineers maintaining software developed by Iress, whose code exists in another version control system (e.g TFS).

  • Product operations who are maintaining the list of Iress Products & Features.


Atlas is a registry of all Iress Products, Features & Software (PFS). This registry allows anyone to explore the relationships, dependencies, key attributes and owners of our PFS.

Atlas also ensures that all code repos are under active custodianship, it will archive any dormant repos.


The Atlas registry is used by a number of downstream systems, which aim to reduce the friction in surfacing operational and security information to relevant stakeholders, e.g TOR Dashboard, Uncaged, AppSec Dashboard

By not registering in Atlas, you lose access to this information, risk your repo being archived, and reduce our ability to monitor incidents and notify software owners.


Engineering Enablement


Registering on Atlas requires you to commit a yaml file into a GitHub repository. The Atlas yaml file has a defined specification that will be validated in your PR.


Gathering the information to populate the Atlas yaml file should take less than four hours.


Making changes to the yaml should take under 10 minutes. The time to merge these changes will be determined by the speed of your repository’s pre-merge checks.
